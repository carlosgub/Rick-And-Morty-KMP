Arquitecturas avanzadas con Kotlin Flow y StateFlow

stream

Flow

Stream frío que emite valores cuando se colecta

Uso:
Operaciones asíncronas, transformaciones de datos
flow { emit(1) delay(1000) emit(2) }.collect { value -> println(value) }
radio_button_checked

StateFlow

Stream caliente con estado actual siempre disponible

Uso:
Estado de UI, ViewModels, configuración
val state = MutableStateFlow( UiState.Loading ) state.value = UiState.Success // Siempre tiene valor
share

SharedFlow

Stream caliente para eventos sin estado inicial

Uso:
Eventos one-time, navegación, toasts
val events = MutableSharedFlow< NavigationEvent>() events.emit( NavigateToDetail(id) )

code Patrones Avanzados

Combinación de Flows

combine( userFlow, settingsFlow ) { user, settings -> UiState(user, settings) }.collect { state -> updateUI(state) }

Debounce para búsqueda

searchQuery .debounce(300) .distinctUntilChanged() .flatMapLatest { query -> repository.search(query) }

Retry con backoff

flow { emit(apiCall()) } .retry(3) { cause -> delay(1000 * attempt) cause is IOException }

Cache con stateIn

repository.getData() .stateIn( scope = viewModelScope, started = SharingStarted.WhileSubscribed(5000), initialValue = null )

tips_and_updates Mejores Prácticas

✓ Usa StateFlow para estado de UI
✓ SharedFlow para eventos one-time
✓ Cancela flows con viewModelScope
✓ Evita collect en el hilo principal
✓ Usa operators para transformaciones
✓ Maneja errores con catch
